(Copy dari: Settings → API)

### 2.2 Create Table

1. **Di Supabase dashboard**, go to: SQL Editor
2. **Click:** "New Query"
3. **Copy-paste SQL ini:**

```sql
-- Create waitlist table
CREATE TABLE waitlist (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  name TEXT,
  role TEXT,
  university TEXT,
  source TEXT,
  ip_address TEXT,
  user_agent TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create index for faster lookups
CREATE INDEX idx_waitlist_email ON waitlist(email);

-- Enable Row Level Security
ALTER TABLE waitlist ENABLE ROW LEVEL SECURITY;

-- Allow public inserts (untuk waitlist form)
CREATE POLICY "Allow public inserts" ON waitlist
FOR INSERT
TO anon
WITH CHECK (true);

-- Allow public reads (untuk stats - optional)
CREATE POLICY "Allow public reads" ON waitlist
FOR SELECT
TO anon
USING (true);
```

4. **Click:** "Run" ▶️
5. **Verify:** Go to Table Editor → Should see `waitlist` table

✅ **CHECK:** Table created!

## ✅ STEP 5: Deploy Backend to Railway (10 menit)

### 5.1 Install Railway CLI

```bash
# Install globally
npm install -g @railway/cli

# Verify
railway --version
```

### 5.2 Login & Initialize

```bash
# Navigate ke project
cd ai-academic-platform/services/waitlist-api

# Login ke Railway
railway login
# Browser akan open → Authorize

# Initialize project
railway init
# Pilih: Create new project
# Name: ai-nusantara-waitlist-api
```

### 5.3 Add Environment Variables

```bash
# Add variables satu per satu
railway variables set NODE_ENV=production
railway variables set PORT=3001
railway variables set SUPABASE_URL=https://xxxxx.supabase.co
railway variables set SUPABASE_ANON_KEY=eyJhbGc...
railway variables set RESEND_API_KEY=re_xxxxx
railway variables set FROM_EMAIL=noreply@ai-nusantara.id
railway variables set FROM_NAME=AI NUSANTARA
railway variables set API_KEY=ai-nusantara-secret-key-2023
railway variables set ALLOWED_ORIGINS=https://your-netlify-url.netlify.app

# Verify
railway variables list
```

**IMPORTANT:** Update `ALLOWED_ORIGINS` setelah deploy landing page (Step 6)

### 5.4 Deploy

```bash
# Deploy ke Railway
railway up

# Wait... (1-2 menit)
# Will show deployment progress
```

### 5.5 Get Production URL

```bash
# Set public domain
railway domain

# Will provide URL seperti:
# https://ai-nusantara-waitlist-api.up.railway.app
```

✅ **SAVE:** `https://ai-nusantara-waitlist-api.up.railway.app`

### 5.6 Test Production Backend

```bash
# Test health endpoint
curl https://ai-nusantara-waitlist-api.up.railway.app/health

# Should return: {"status":"healthy",...}
```

✅ **CHECK:** Backend live!

## 🆘 TROUBLESHOOTING

### Problem: CORS Error
**Symptom:** Browser console shows "CORS policy error"
**Fix:**
```bash
# Update ALLOWED_ORIGINS dengan exact landing page URL
railway variables set ALLOWED_ORIGINS=https://exact-url.netlify.app
railway up
```

### Problem: Email Not Received
**Symptom:** Form submits tapi no email
**Fix:**
1. Check Resend dashboard → Logs
2. Check Recipient email ada di logs
3. Check Spam folder
4. Verify FROM_EMAIL domain

### Problem: Database Error
**Symptom:** API returns 500 error
**Fix:**
1. Check Supabase credentials di Railway variables
2. Verify table exists in Supabase
3. Check Railway logs

### Problem: Landing Page Not Updated
**Symptom:** API URL still localhost
**Fix:**
1. Edit `landing.html` line ~806
2. Save file
3. Re-deploy ke Netlify (drag & drop lagi)
