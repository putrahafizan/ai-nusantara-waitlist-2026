## 🔧 PART 2: Setup Supabase (10 menit)

### Kenapa Supabase?
- ✅ FREE tier (500MB database)
- ✅ No credit card required
- ✅ Easy setup
- ✅ Built-in authentication (for later)
- ✅ Great dashboard

### Steps:

1. **Create Account**
   - Go to: https://supabase.com
   - Sign up with GitHub

2. **Create Project**
   - Click "New Project"
   - Name: `ai-nusantara`
   - Database Password: (generate & save!)
   - Region: Singapore (closest to Indonesia)
   - Click "Create new project"
   - Wait ±2 minutes

3. **Get Credentials**
   - Go to: Project Settings > API
   - Copy:
     - Project URL
     - anon public key

4. **Create Table**

   Go to: SQL Editor > New Query

   ```sql
   -- Create waitlist table
   CREATE TABLE waitlist (
     id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
     email TEXT NOT NULL UNIQUE,
     name TEXT,
     role TEXT,
     university TEXT,
     source TEXT,
     ip_address TEXT,
     user_agent TEXT,
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
   );

   -- Create index for faster lookups
   CREATE INDEX idx_waitlist_email ON waitlist(email);

   -- Enable Row Level Security (optional, for security)
   ALTER TABLE waitlist ENABLE ROW LEVEL SECURITY;

   -- Allow public inserts (for waitlist form)
   CREATE POLICY "Allow public inserts" ON waitlist
   FOR INSERT
   TO anon
   WITH CHECK (true);
   ```

5. **Save Credentials**
   - Save URL & key somewhere safe
   - You'll need them for .env

## 🚀 PART 4: Deploy Backend API (15-20 menit)

### Option A: Railway (Recommended - FREE tier)

1. **Install Railway CLI**
   ```bash
   npm install -g @railway/cli
   ```

2. **Login**
   ```bash
   railway login
   ```

3. **Create Project**
   ```bash
   cd ai-academic-platform/services/waitlist-api
   railway init
   ```

4. **Add Variables**
   ```bash
   # Add environment variables
   railway variables set NODE_ENV=production
   railway variables set PORT=3001
   railway variables set SUPABASE_URL=your-supabase-url
   railway variables set SUPABASE_ANON_KEY=your-supabase-key
   railway variables set RESEND_API_KEY=your-resend-key
   railway variables set FROM_EMAIL=noreply@ai-nusantara.id
   railway variables set ALLOWED_ORIGINS=https://your-landing-page-url.netlify.app
   railway variables set API_KEY=generate-random-string-here
   ```

5. **Deploy**
   ```bash
   railway up
   ```

6. **Get URL**
   - Railway will provide URL like: `https://your-project.railway.app`
   - Copy this URL

### Option B: Render (FREE tier)

1. **Prepare render.yaml**
   - File already in repo

2. **Connect to Render**
   - Go to: https://render.com
   - Sign up
   - Click "New +" > "Web Service"

3. **Connect GitHub**
   - Connect your repo
   - Select `services/waitlist-api`

4. **Configure**
   - Name: `ai-nusantara-waitlist-api`
   - Environment: Node
   - Build Command: `npm install`
   - Start Command: `npm start`

5. **Add Environment Variables**
   In Render dashboard, add:
   ```
   NODE_ENV=production
   PORT=3001
   SUPABASE_URL=your-url
   SUPABASE_ANON_KEY=your-key
   RESEND_API_KEY=your-key
   ALLOWED_ORIGINS=your-landing-url
   API_KEY=random-string
   ```

6. **Deploy**
   - Click "Deploy"
   - Wait ±5 minutes
   - Get URL

### Option C: Fly.io (FREE tier)


## 🔧 PART 8: Troubleshooting

### Issue: CORS Error

**Symptom:** Browser console shows CORS policy error

**Fix:**
```env
# Make sure landing page URL is in ALLOWED_ORIGINS
ALLOWED_ORIGINS=https://your-landing-page.netlify.app,https://www.custom-domain.com
```

### Issue: Email Not Sending

**Symptom:** User submits but no email received

**Check:**
1. Resend dashboard > Logs
2. API server logs
3. Spam folder

**Fix:**
- Verify RESEND_API_KEY
- Check FROM_EMAIL is verified
- Check recipient email is valid

### Issue: Database Error

**Symptom:** API returns 500 error

**Check:**
1. Supabase credentials in .env
2. Table exists
3. Connection logs

**Fix:**
- Recreate table if needed
- Regenerate API keys
- Check network/firewall

### Issue: Duplicate Emails

**Symptom:** Same email can be submitted twice

**Check:**
1. Database unique constraint
2. API validation logic

**Fix:**
- Ensure UNIQUE constraint on email column
- Check validation in waitlistService.js
