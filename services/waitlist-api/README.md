# Waitlist API - AI NUSANTARA

Minimal backend API untuk landing page waitlist functionality.

## 🚀 Fitur

- ✅ Email validation & sanitization
- ✅ Duplicate email detection
- ✅ Supabase integration (or local file fallback)
- ✅ Email confirmation via Resend
- ✅ Rate limiting
- ✅ CORS configuration
- ✅ Error handling
- ✅ Stats endpoint

## 📦 Prerequisites

- Node.js 18+ installed
- (Optional) Supabase account for database
- (Optional) Resend account for emails

## 🔧 Quick Start

### 1. Install Dependencies

```bash
cd ai-academic-platform/services/waitlist-api
npm install
```

### 2. Setup Environment Variables

```bash
cp .env.example .env
```

Edit `.env`:
```env
NODE_ENV=development
PORT=3001

# Supabase (optional but recommended)
SUPABASE_URL=your-supabase-url
SUPABASE_ANON_KEY=your-supabase-key

# Resend (optional)
RESEND_API_KEY=your-resend-key
FROM_EMAIL=noreply@ai-nusantara.id

# CORS
ALLOWED_ORIGINS=http://localhost:8000,https://your-landing-page-url.com
```

### 3. Start Server

**Development:**
```bash
npm run dev
```

**Production:**
```bash
npm start
```

Server will run on `http://localhost:3001`

## 📊 API Endpoints

### POST /api/waitlist
Add email to waitlist

**Request:**
```json
{
  "email": "user@example.com",
  "name": "John Doe",
  "role": "dosen",
  "university": "Universitas Indonesia"
}
```

**Response (201 Created):**
```json
{
  "success": true,
  "message": "Berhasil bergabung dengan waitlist!",
  "data": {
    "email": "user@example.com",
    "createdAt": "2026-03-03T10:00:00.000Z"
  }
}
```

**Error (409 Conflict):**
```json
{
  "error": "Email already registered",
  "code": "EMAIL_EXISTS",
  "message": "Terima kasih! Email Anda sudah terdaftar."
}
```

### GET /api/waitlist/stats
Get waitlist statistics (requires API key)

**Request:**
```bash
curl -H "X-API-Key: your-api-key" http://localhost:3001/api/waitlist/stats
```

**Response:**
```json
{
  "success": true,
  "data": {
    "total": 150,
    "byRole": {
      "dosen": 80,
      "mahasiswa": 60,
      "peneliti": 10
    },
    "bySource": {
      "landing-page": 120,
      "linkedin": 20,
      "whatsapp": 10
    },
    "latestSignups": [...]
  }
}
```

## 🗄️ Database Setup

### Option 1: Supabase (Recommended)

1. **Create Supabase Project:**
   - Go to https://supabase.com
   - Create new project
   - Get your URL and anon key

2. **Create Table:**
```sql
CREATE TABLE waitlist (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  name TEXT,
  role TEXT,
  university TEXT,
  source TEXT,
  ip_address TEXT,
  user_agent TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create index for faster lookups
CREATE INDEX idx_waitlist_email ON waitlist(email);
```

3. **Update .env:**
```env
SUPABASE_URL=your-project-url
SUPABASE_ANON_KEY=your-anon-key
```

### Option 2: Local File Storage (No Database)

If you don't want to setup Supabase, the API will automatically fallback to storing emails in `services/waitlist-api/data/waitlist.json`.

```env
USE_LOCAL_STORAGE=true
```

## 📧 Email Setup

### Option 1: Resend (Recommended)

1. **Create Resend Account:**
   - Go to https://resend.com
   - Get API key (FREE 100 emails/day)

2. **Verify Domain:**
   - Add your domain in Resend dashboard
   - Update DNS records

3. **Update .env:**
```env
RESEND_API_KEY=re_xxxxx
FROM_EMAIL=noreply@ai-nusantara.id
FROM_NAME=AI NUSANTARA
```

### Option 2: No Email (Development Mode)

If you don't setup Resend, emails will be logged to console in development mode.

## 🚀 Deployment

### Option 1: Railway (Easiest - FREE tier available)

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Initialize project
cd services/waitlist-api
railway init

# Deploy
railway up
```

### Option 2: Render (FREE tier available)

1. Create `render.yaml`:
```yaml
services:
  - type: web
    name: ai-nusantara-waitlist-api
    env: node
    buildCommand: npm install
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 3001
```

2. Connect GitHub repo to Render
3. Deploy automatically on push

### Option 3: Vercel/Netlify

1. Install `serverless-http`:
```bash
npm install serverless-http
```

2. Create `api/waitlist.js`:
```javascript
const serverless = require('serverless-http');
const app = require('../../services/waitlist-api/server');

module.exports = serverless(app);
```

3. Deploy to Vercel/Netlify

## 🐛 Troubleshooting

**Issue: CORS errors**
- Check `ALLOWED_ORIGINS` in .env
- Make sure landing page URL is included

**Issue: Email not sending**
- Check Resend API key
- Check console logs for errors
- Verify domain is configured in Resend

**Issue: Database not working**
- Check Supabase credentials
- Verify table exists
- Check connection logs
